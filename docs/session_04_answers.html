Answer

Answer

mutate()

Answer

summarize()

Answer

  • Collapse many values into a single summary

What are the three “rules” of dplyr verbs?

Answer

dplyr functions are written as verbs with the following rules:

  1. The first argument is a data frame.

  2. Inside the function, you can use column names as if they were variables.

  3. The output is a new data frame.

Review Task

Use the pipe operator %>% and dplyr verbs to complete the following task:

  1. Use the example dataset

  2. Filter out tumors larger than 10

  3. Calculate follow_up time as the number of years between diagnosis and the patient’s visit

  4. Calculate the average follow up time by site_code

Answer

example %>% 
  filter(tumor_size <= 10) %>% 
  mutate(follow_up = age_visit - age_dx) %>% 
  group_by(site_code) %>% 
  summarize(avg_follow_up = mean(follow_up))
# A tibble: 3 x 2

Importing Data

Gather your “exported” data

Download the CancerRegistryDataPatients.xlsx and CancerRegistryDataPatients.csv data files from the Session 4 Materials page.

In the Files pane of your RStudio project, click More ▸ Show Folder in New Window… to open the folder in a file browser. Copy or move the data files from your downloads folder into your project folder. Check that you see the files in RStudio.

If you’re running RStudio in your browser (e.g. on rstudio.cloud), you can run the download.file() commands in the materials page.

Import Using RStudio

There are two main packages for reading in data: readr for CSV and readxl for Excel.

If you have the option in your data source, choose to export your data as a CSV file. The options and control for importing into R using readr are more powerful and more easily customized.

Import Excel File

Import Code

library(readxl)
cr_patients <- read_excel("CancerRegistryDataPatients.xlsx",
                          col_types = c("numeric", "numeric", "text",
                                        "text", "text", "text", "date", "text"))

Import CSV File

Import Code

library(readr)
cr_patients <- read_csv("CancerRegistryDataPatients.csv", 
    col_types = cols(`Date Last Patient Contact/Dead` = col_date(format = "%m/%d/%y")))

A Note About Dates

Here is a small reference for date/time formats. See ?strftime for more time format placeholders.

  • %m/%d/%y: 01/02/06

  • %m/%d/%Y: 01/02/2006

  • %Y-%m-%d: 2006-01-02

  • %m/%d/%y %H:%M: 01/02/06 15:04

  • %m/%d/%y %H:%M:%S: 01/02/06 15:04:05

  • %m/%d/%y %I:%M:%S %p: 01/02/06 03:04:05 PM

Export The Data

Export the data using write_csv()

To export the data, use the function write_csv() from readr. This function takes the data frame as the first object, and a path (or filename) where the file should be written.

write_csv(cr_patients_export, "CancerRegistryPatients_export.csv")

It’s a good idea to save your output files in a separate directory, like output_04 for outputs from this session. If you get an error like this

Warning in open.connection(path, "wb"): cannot open file 'output_04/
CancerRegistryPatients_export.csv': No such file or directory
Error in open.connection(path, "wb"): cannot open the connection

it is most likely because the directory output_04 doesn’t exist in your current working directory. You can use the Files pane in RStudio to create one, or you can run

dir.create("output_04")
